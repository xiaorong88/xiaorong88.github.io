菜鸟Hexo安装→建站→博客书写→部署到github

【安装环境：windows 10 （64位）】

  • hexo之 API

  • 安装工具：

    • 安装Git：下载安装包

    • 安装nodejs：下载安装包（最新版本自带npm模块）

    • npm的安装。由于新版的NodeJS已经集成了npm，所以之前npm也一并安装好了。

      ​

  • 配置npm与环境变量

    • 配置npm的全局模块的存放路径以及cache的路径。

    • 将以上两个文件夹放在NodeJS的主目录下（我的nodejs默认安装在C:\Program Files\nodejs）

    • 在NodeJs下建立”node_global”及”node_cache”两个文件夹

      ​

      ​

    • 启动cmd，输入

    • 环境变量配置

      1. “我的电脑”右键“属性”-“高级系统设置”-“高级”-“环境变量”。
      2. 进入环境变量对话框，在系统变量下新建”NODE_PATH”，输入”C:\Program Files\nodejs\node_global\node_modules“。（ps：这一步相当关键。）
      3. 2014.4.19新增：由于改变了module的默认地址，所以上面的用户变量都要跟着改变一下（用户变量”PATH”修改为“C:\Program Files\nodejs\node_global\”），要不使用module的时候会导致输入命令出现“xxx不是内部或外部命令，也不是可运行的程序或批处理文件”这个错误。

  • 使用npm安装hexo模块

    • 管理员权限运行“命令提示符” 然后 npm install -g hexo-cli

      • 安装完成后，在C:\Program Files\nodejs\node_global\node_modules目录下会出现hexo-cli文件夹及文件夹中的相关文件

        ​

    • 再次开启 管理员权限运行“命令提示符”，进入node，输入require('hexo-cli')来测试下node的模块全局路径是否配置正确了。正确的话cmd会列出hexo-cli的相关信息

    • ​

  • 建站，初始化

    • 管理员权限运行“命令提示符” 然后 hexo init [folder] ,如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站
    • 我创建在D:\Study\hexo，生成的文件，如下图：

  • 站点中新建一篇文章

    • 管理员权限运行“命令提示符” 然后 hexo new [layout] <title> ,如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来

    • 我的新建文章：hexo new “菜鸟Hexo安装到建站”，在D:\Study\hexo\source_posts中会创建一个文件：菜鸟Hexo安装到建站.md

  • 编辑文章（菜鸟Hexo安装到建站.md)

    • 使用工具：Typora下载地址
    • 使用markdown语法：https://www.zybuluo.com/mdeditor?url=https://www.zybuluo.com/static/editor/md-help.markdown
